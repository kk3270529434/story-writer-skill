# AI 味检测与清除指南

> 更新日志：2026-06-11 | v1.0 | 基于大量 AI 中文写作特征提炼
> 使用方法：生成故事后，逐项对照检测。每命中一项扣一分，扣满 5 分即判定为"AI 味过重"，需重写。

---

## 一、句式层 AI 味（最易检测）

### 1.1 对称句式滥用
AI 写中文最爱造对仗——但真正的茶馆叙事是"散"的，不是"整"的。

| ❌ AI 味对称句 | ✅ 茶馆破对称 |
|---------------|-------------|
| "夜，深了。风，起了。" | "到这会儿天已经黑透了。" |
| "有人说是巧合，有人说是宿命。" | "到底怎么回事，谁也说不清楚。" |
| "恐惧如同潮水，吞没了他最后的理智。" | "他吓坏了。就这么简单。" |
| "那是一个寒冷的冬夜，北风呼啸着穿过空旷的街道。" | "十二月底，北京下了一场大雪。" |

**检测方法**：搜索连续两个结构相同的短句（字数相近、词性对应）→ 标记为疑似 AI。

### 1.2 过度完整的叙事弧
AI 喜欢"起承转合"完整无缺。茶馆故事可以有"毛边"——一个伏笔没收、一个角色没交代清楚、一段背景跳过不表。

| ❌ AI 味完整弧 | ✅ 茶馆毛边 |
|---------------|-----------|
| 所有伏笔都回收 | 留 1-2 个"未解之谜" |
| 每个角色都有结局 | 配角可以突然不再出现 |
| 背景解释无遗漏 | 可以说"具体为啥我也不知道" |
| 时间线精确到分钟 | 可以说"大概是那几天的事" |

### 1.3 对称式转折词
AI 高频使用对称式转折："一方面……另一方面……""既……又……""不是……而是……"

**检测**：搜索 `一方面` `另一方面` `既……又` `不是……而是` `无论……都` —— 出现即删。

## 二、词汇层 AI 味

### 2.1 AI 高频形容词（"虚词库"）
以下词汇在 AI 生成文本中出现频率远超人类写作：

```
一级（绝禁）：悄然 蓦然 倏地 璀璨 绚丽 静谧 安详 弥漫 笼罩 涌起 不禁 不由得
二级（高危）：渐渐 缓缓 深深 默默 微微 淡淡 远远 轻轻
三级（警惕）：似乎 仿佛 宛如 犹如 若隐若现 若有所失
```

**检测脚本思路**：
```python
ai_adj = ['悄然','蓦然','倏地','璀璨','绚丽','静谧','安详','弥漫','笼罩','涌起','不禁','不由得']
count = sum(text.count(w) for w in ai_adj)
# count > 2 → 警告
```

### 2.2 AI 高频虚词
AI 喜欢在句首加"其实""只是""或许""大概"来模仿人类犹豫感——但用得太多就暴露了。

```
其实 → 每 2000 字最多 1 次
只是 → 同上
或许/也许/大概 → 三选一，每篇最多 2 次
```

### 2.3 书面化程度检测
数一数"的"字密度。人类口语中"的"字密度低于书面语。

- 书面语："他是一个有着丰富经验的老刑警"
- 茶馆口语："他干刑侦干了二十多年"

**快速检测**：`的` 字占总字数比例 > 4% → 书面语过重。

## 三、结构层 AI 味

### 3.1 完美段落长度
AI 倾向于每个段落长度均匀（3-5 句，工工整整）。人类写作有长有短。

| ❌ AI 味均匀段落 | ✅ 人类不规则段落 |
|-----------------|-----------------|
| 段段 3-4 行 | 有 1 行段、有 8 行段、然后又是 1 行段 |
| 每段都有完整的"开始-展开-收束" | 有的段没说完就断了下段接 |
| 段与段之间有明显的逻辑连接词 | 段与段之间靠空行和意合 |

### 3.2 情绪均匀分布
AI 倾向于让"恐怖/搞笑/温情"均匀交替。人类会集中在某些区域爆发。

**检测**：统计笑点间隔。如果每 500 字恰好一个笑点——太均匀了，AI。真正的茶馆：有一段连续 800 字没笑点（紧绷），然后连着 200 字全是包袱（释放）。

### 3.3 对话密度过于均匀
AI 写对话会让每个角色发言长度相近。人类：有人话多，有人沉默。

**检测**：统计角色 A 和角色 B 的平均话轮长度。如果相差 < 20% → AI 味。

## 四、内容层 AI 味

### 4.1 无具体数字/品牌/地名
AI 喜欢用泛指："某个小镇""一家餐厅""一辆旧车"。人类会不自觉地给出具体细节。

| ❌ AI 味泛化 | ✅ 人类细节 |
|-------------|-----------|
| 某个小镇 | 河南修武县河湾村 |
| 一家路边餐厅 | 杨国福大酒店（麻辣烫） |
| 一辆旧车 | 九几年的长城皮卡 |
| 一个老警察 | 县刑侦大队副大队长范新河 |

**检测**：搜索故事中的具体地名/品牌/人名/年份/价格。如果全是泛指 → AI 味极重。

### 4.2 "所有人都""每个人都在""没人能"
绝对化表达是 AI 的一个标志。

| ❌ 禁止 | ✅ 替换 |
|--------|--------|
| 所有人都在 | 村里人 / 哥几个 / 我们那帮人 |
| 没人能理解 | 连他自己都想不明白 |
| 永远 | 到死 / 一辈子 |
| 一切都 | 这事儿 / 这些事 |

### 4.3 情感标签直给
AI 喜欢直接告诉读者角色的情感状态。人类通过行为和对话间接传达。

| ❌ AI 直给 | ✅ 茶馆间接 |
|-----------|-----------|
| 他感到一阵恐惧 | 他后背一凉 |
| 气氛变得诡异起来 | 谁也不说话了 |
| 他的眼中充满了悲伤 | 他把照片翻过来扣在桌上 |
| 那是一个令人毛骨悚然的夜晚 | 那天晚上，老姨自己走夜路 |

## 五、自动检测脚本

以下 Python 脚本可在生成故事后自动扫描 AI 味指标：

```python
def detect_ai_flavor(text: str) -> dict:
    """返回 AI 味检测报告"""
    
    # 一级禁令词
    forbidden = ['悄然','蓦然','倏地','璀璨','绚丽','静谧','安详',
                 '弥漫着','笼罩在','心中涌起','不禁','不由得','情不自禁地']
    hits = {w: text.count(w) for w in forbidden if w in text}
    
    # 对称句式检测
    import re
    parallel_pattern = re.findall(r'([^。！？\n]{4,12})，([^。！？\n]{4,12})。', text)
    
    # 的密度检测
    de_ratio = text.count('的') / max(len(text.replace('\n','')), 1)
    
    # 虚词密度
    xu_words = ['其实','只是','或许','也许','大概','似乎','仿佛','宛如']
    xu_count = sum(text.count(w) for w in xu_words)
    
    report = {
        '一级禁令命中': hits,
        '对称句式数': len(parallel_pattern),
        '的字密度': f'{de_ratio:.2%}',
        '虚词密度': f'{xu_count / max(len(text), 1) * 1000:.1f}‰',
        '判定': 'PASS' if (len(hits) == 0 and de_ratio < 0.04 and len(parallel_pattern) < 3) else 'WARN'
    }
    return report
```

## 六、AI 味修复优先级

| 优先级 | 问题 | 修复方法 |
|:------:|------|----------|
| P0 | 一级禁令词 | 直接删除，或用口语替换 |
| P0 | 对称句式 | 打碎其中一半，拉长或缩短 |
| P1 | 的字密度 > 4% | 减少定语，用动词代替形容词 |
| P1 | 无具体细节 | 加至少 3 个具体地名/品牌/数字 |
| P2 | 段落均匀 | 合并 2-3 段 + 拆散 1 段 |
| P2 | 情绪均匀 | 合并几个笑点到一个区域 |
| P3 | 对话均匀 | 让一个角色沉默，让另一个絮叨 |

## 七、与 forbidden-phrases.md 的分工

| 文件 | 职责 |
|------|------|
| `forbidden-phrases.md` | 词汇/句式级黑名单（**写作时避免**） |
| `ai-detection.md`（本文件） | 结构/模式级检测（**写完后扫描**） |

两者配合使用：写的时候参照 forbidden-phrases 避开禁忌词，写完了用 ai-detection 扫描结构问题。
