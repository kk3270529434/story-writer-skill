# 深夜小茶馆风格 — 短篇故事创作技能

## 核心指令

当用户提供创作需求时，按以下螺旋流程执行（允许回溯修改）：

### 1. 确认创作意图

使用 `AskUserQuestion` 工具向用户确认以下维度。**每个维度必须提供具体选项**（不要只给抽象分类），选项附带场景说明以便用户理解差异。用户已提供的信息不再重复询问。

#### 1.1 必问维度（每次创作都需确认）

**A. 故事主题** — 决定风格调性和结构骨架

| 选项 | 说明 | 代表作品 |
|------|------|----------|
| 撞鬼/凶宅 | 主角在特定地点遭遇灵异，恐惧→对抗→揭秘 | 全国八大凶宅系列 |
| 精怪/仙家 | 涉及胡黄白柳灰等民间信仰，半人半妖 | 东北保家仙、灰仙 |
| 都市怪谈 | 当代城市背景的诡异传闻，强调"真实感" | 午夜外卖、停车场惊魂 |
| 因果报应 | 先果后因的倒叙结构，贪念→报应→悔悟 | 索命钱、夺命传真 |
| 民间传说 | 依托历史事件/地方风俗展开的非虚构感叙事 | 营口坠龙、上海龙柱 |
| 奇术秘法 | 涉及风水/道术/蛊术等专业知识 | 茅山道长斗法、衡阳害手 |

**B. 篇幅** — 决定章节数和节奏

| 选项 | 字数 | 章节 | 适用场景 |
|------|------|:----:|----------|
| 单集短篇 | 2000-3500字 | 3-5节 | 单一事件、一个转折即可讲完 |
| 上下集 | 4000-6000字 | 每集3-5节 | 有历史纵深、需要铺垫的复杂故事 |
| 长篇连载 | 6000-10000字 | 3-4集 | 大型民间传说、多线叙事 |

**C. 年代背景** — 决定语言风格、道具细节、社会语境

| 选项 | 时间范围 | 标志性细节 |
|------|----------|------------|
| 清代/民国 | 1900-1949 | 长袍马褂、黄包车、电报、租界 |
| 建国初期 | 1950-1978 | 工分、粮票、公社、大炼钢铁、知青 |
| 改革开放 | 1979-1999 | BB机、自行车大国、南下打工、大学扩招前 |
| 当代都市 | 2000-至今 | 智能手机、外卖、网约车、社交媒体 |

**D. 地域** — 决定方言口吻、民俗参考、环境描写

| 选项 | 方言特色 | 典型民俗/场景 |
|------|----------|---------------|
| 东北 | 嘛/咋/啥/甭/俺/整 | 保家仙、农村火炕、林场、油田 |
| 华北（京津） | 嘛/您/倍儿/甭介/怎么茬儿 | 四合院、故宫、老胡同、高校 |
| 华东（沪杭） | 侬/阿拉/好得很/不要紧的 | 弄堂、龙柱、城隍庙、西湖 |
| 华南（粤闽） | 冇/嘅/啦/衰/阴功 | 骑楼、祠堂、荔湾、城中村 |
| 西南（川渝） | 撒/哈/安逸/巴适/恼火 | 山城、茶馆、三峡、苗寨 |

**E. 主角类型** — 决定叙事视角和读者代入感

| 选项 | 典型职业 | 性格特征 |
|------|----------|----------|
| 城市打工人 | 外卖员、网约车司机、保安、服务员 | 底层视角，为生计奔波 |
| 职场白领 | 程序员、设计师、公司职员 | 受过教育，理性→被颠覆 |
| 在校学生 | 大学生、研究生 | 好奇心强，容易犯忌讳 |
| 边缘闲散者 | 古玩贩子、社会盲流、无业者 | 灰色地带，认识三教九流 |
| 普通居民 | 退休老人、家庭主妇、小区住户 | 被动卷入，代表"普通人" |
| 专业人士 | 医生、教师、警察、军人 | 特殊身份带来独特视角 |

#### 1.2 可选维度（根据主题选问）

**F. 恐怖程度**（撞鬼/凶宅类建议询问）

| 选项 | 说明 |
|------|------|
| 细思极恐 | 以心理恐怖为主，不做血腥描写 |
| 直面惊悚 | 有详细灵异场景描写，紧张感强 |
| 轻松灵异 | 恐怖与搞笑各半，适合胆小读者 |

**G. 叙事人称**（如用户未指定则默认框架式）

| 选项 | 说明 |
|------|------|
| 框架式（默认） | 杨湃茶馆叙事 + 第三人称故事主体 |
| 纯第一人称 | "我"亲历的恐怖事件 |
| 纯第三人称 | 完全旁观叙事，叙述者不介入 |

**H. 结尾风格**

| 选项 | 说明 |
|------|------|
| 温情收束（默认） | 恐怖之后有温暖/哲理，符合茶馆调性 |
| 恐怖留白 | 戛然而止，不给解释，让读者细思极恐 |
| 幽默翻转 | 结尾抖一个搞笑包袱，消解恐怖 |

**I. 是否需要民俗考据**
- 是：故事中涉及的专业民俗知识（如出马仙仪式、茅山法术等）需要详细展开，作为故事的知识点
- 否（默认）：民俗知识点到为止，以叙事为主

**J. 是否有额外素材提供**
- 是：用户有新闻链接、亲身经历、听来的传闻、旧照片等素材，希望改编融入故事
  - 请用户直接粘贴或描述素材内容
  - 素材的类型可能是：新闻事件（社会案件、灾难事故、奇闻异事）、个人/朋友亲历、家族口传、旧报纸/档案片段、特定地点/建筑物的背景
  - 获取素材后，用 2-3 句话向用户确认"你打算如何将素材融入情节骨架"
- 否（默认）：完全原创构思

#### 1.3 问法规范

**原则**：
- 每次用 `AskUserQuestion` 最多问 4 个维度
- 如果用户已明确给出某个维度的信息，跳过不问
- 选项标签要简洁（≤12字），描述要有场景感
- 第一个选项标注"(推荐)"，引导用户快速决策

**示例**：用户说"给我写个校园恐怖故事"
```
已确认：主题=撞鬼/凶宅，主角=在校学生
待确认：篇幅、年代、地域、恐怖程度
→ 用 AskUserQuestion 问这4个维度
```

#### 1.4 意图确认完成标志
- 所有必问维度已获得用户答复
- 可选维度中与主题强相关的已确认
- 向用户回显一份"创作卡"摘要，格式如下：

```
📝 创作卡
━━━━━━━━━━━━━━━━━━
主题：都市怪谈
篇幅：单集短篇（2500-3500字）
年代：90-00年代
地域：华北（北京）
主角：在校大学生
基调：细思极恐 + 温情收束
风格：杨湃第一人称框架叙事
━━━━━━━━━━━━━━━━━━
```

### 1.5 素材整合与年代考据

> 此步骤在确认创作意图后、"选择情节骨架"前执行。目的是将用户素材和年代背景知识预先准备好，供后续创作调用。

#### A. 整合用户素材（如有）

当用户在"J. 是否有额外素材提供"中选择了"是"：

1. **接收素材**：让用户自由描述或粘贴（可以是新闻链接、口述、片段记忆等）
2. **提取关键要素**：从素材中抽取以下可改编要素并回显给用户确认：
   ```
   素材关键要素：
   - 核心事件：[一句话概括]
   - 时间/地点：[如素材中有]
   - 关键人物：[如素材中有]
   - 可改编的"钩子"：[素材中最吸引人的点]
   - 需要虚构成分：[哪些空白需要创作填充]
   ```
3. **改编策略确认**：
   - 忠实改编：保留事件主干，替换人名地名，增加灵异元素
   - 灵感取用：仅取素材中的一个概念/场景作为创作起点，其余自由发挥
   - 嵌套引用：素材作为"故事中某人讲述的故事"出现（降低真实性风险）
4. **素材融入位置标记**：在后续提纲中标注素材将用于哪个章节/情节节点

**注意**：如果用户素材涉及真实案件或敏感事件，应建议使用"嵌套引用"策略，并在故事中添加"本故事纯属虚构"的隐含表达。

#### B. 年代背景考据（互联网搜索）

当用户选择的年代早于 2000 年，或故事需要嵌入特定年代氛围时，**必须执行此步骤**。

**触发条件**：
- 年代设定在 2000 年以前
- 故事中需要提到当时的流行文化、物价、社会事件作为"年代锚点"
- 面向年轻读者，需要解释已消失的事物（如粮票、工分、BB机等）

**搜索策略** — 分三路并行搜索：

| 搜索方向 | 搜索词示例 | 用途 |
|----------|-----------|------|
| **流行文化** | `[年份]年 流行歌曲` `[年代]经典老歌` | 在故事中作为环境音/角色哼唱出现，增强代入感 |
| **社会趣闻** | `[年份]年 趣闻` `[年代]老百姓生活` `[年份]物价` | 叙述者穿插解释时代背景时引用，帮助年轻读者理解 |
| **名人轶事** | `[年份]年 [地域]名人` `[年代][行业]人物趣事` | 作为叙述者的"知识彩蛋"，如"那时候XXX刚红，满大街都放他的歌" |

**实战示例**：
```
年代设定：1983年
地域：东北农村

搜索1："1983年流行歌曲" → 《我的中国心》《乡恋》《在希望的田野上》
  → 用法：故事中老姨放学路上哼《乡恋》，被坟地里的声音打断

搜索2："1983年 东北 老百姓生活" → 粮票还没完全取消、自行车结婚三大件
  → 用法：叙述者插入解释"那会儿村里谁家有辆永久，比现在开宝马还牛"

搜索3："1983年 趣闻" → 第一届春晚播出、二王追捕事件
  → 用法：宿舍卧谈会时提及，锚定年代感
```

**考据输出格式** — 在输出提纲前，先展示考据卡片：

```
📜 年代考据卡
━━━━━━━━━━━━━━━━━━
年份：1983年 | 地域：东北
━━━━━━━━━━━━━━━━━━
🎵 流行歌曲：《乡恋》李谷一、《我的中国心》张明敏
📰 社会背景：粮票末期、自行车三大件、春晚元年
💬 可用趣闻：第一届春晚热线被打爆、二王全国通缉
📖 叙述者科普点：
  - "那会儿打电话得去邮局，半夜哪有手机这玩意"
  - "永久牌自行车，相当于现在的奔驰"
━━━━━━━━━━━━━━━━━━
```

**使用规范**：
- 每篇故事使用 2-3 个年代锚点为宜，过多则像百度百科
- 年代细节必须通过**角色行为/环境/对话**自然呈现，禁止叙述者直接罗列
- 叙述者科普年代背景时，用"好多年轻朋友可能不知道……"的口吻，保持茶馆聊天感
- 所有搜索到的信息必须**改写为口语**，禁止直接复制百科原文

### 1.6 设定自检（新增——动笔前强制）

> **解决"核心设定模糊""逻辑矛盾""留白失衡""太像小说"四大问题。必须在输出情节提纲前完成。**

**A. 核心规则一句话**
用一句话写下本故事的超自然核心规则。必须具体、可验证。

| ❌ 模糊 | ✅ 清晰 |
|--------|--------|
| "黄河底下有东西" | "黄河中淹死之人，若背上被刻'送河神符'，尸体三年不腐——刻符者可用其替自己挡一次死劫" |
| "电话能预知未来" | "电话每接通一次，接听者可避开一次危险——代价是寿命缩短一年" |

**B. 规则贯穿检查**：逐节标注——是否有违反规则处？有没有为制造转折而偷改规则？

**C. 留白与解释平衡**
- 必须解释的（≤70%）：规则的基本运作、违反的后果
- 永远不解释的（≥30%）：力量的终极来源、鬼魂的完整动机、"河底到底是什么"
- 标记故意留白处，确保全篇不"说漏嘴"

**D. 去小说化检查（新增）**
- [ ] 故事核心是"茶馆聊天"口吻——不是"小说叙事"口吻？
- [ ] 有无小说特有结构痕迹（"第一章""与此同时，在远方……""镜头切换到……"）？
- [ ] 叙述者是否频繁介入（批讲/现挂/砸挂）——而非隐身让"故事自己推进"？
- [ ] 如果去掉"大家好，我是杨湃"开场——故事还能否成立？去掉就散架→太"小说化"
- [ ] 参考小说素材是否被"**改编**"而非"**搬运**"？法术体系可用，人物关系和情节结构必须原创
- [ ] 故事读起来像不像"一个人在茶馆里跟你聊天"——而不是"出版小说被朗读出来"？

**判定**：若"去小说化"检查有 2 项以上未通过 → 退回重新构思。

### 2. 选择情节骨架

加载 `references/plot-structures.md`。
如有用户素材，额外加载素材关键要素；如有年代考据结果，将锚点融入各章节规划。
加载 `assets/novel-reference.md`（如涉及法术/民俗）——注意：**仅供技术参考，禁止搬运情节**。

根据以上综合信息匹配结构骨架，输出简洁提纲（3-5行）：

```
类型：[撞鬼/精怪/历史/案件/报应]
结构：[单集3幕 / 上下集]
章节规划：#01[功能] → #02[功能] → #03[功能] → #04[功能]
核心冲突：[显性] + [隐性]
关键转折：[2-3个转折点简述]
```

### 3. 试写与即时验证
- 撰写开场200字（从标题到 #01 之前的部分）
- **自检清单**：
  - □ 是否符合 `references/plot-structures.md` 中的开头模式之一？
  - □ 语言指标是否与 `references/style-guide.md` 的量化区间一致（句长15-40字为主，段落1-5行）？
  - □ 是否使用了口语化开场（不是书面化"在遥远的"）？
  - □ 是否包含了 `assets/forbidden-phrases.md` 中的禁用词？
- 如偏离立即修正再继续。

### 4. 分步创作（可回溯）
按 **"框架开场 → #01引入 → #02展开 → #03冲突升级 → #04高潮/揭秘 → 收束 → END → 框架收尾"** 的顺序写作。

每个阶段完成后：
1. 对照 `references/style-guide.md` 检查语言指标
2. 对照 `assets/forbidden-phrases.md` 扫描禁用词
3. 对照 `assets/dialogue-examples.md` 检查对话格式
4. 不符合则局部重写；如需前设伏笔，允许退回修改已完成段落

**关键节点检查**：
- `#01` 结束：是否埋下了至少一个悬念/疑问？
- `#02` 结束：是否有至少一处情绪冲撞（恐怖→搞笑 或 紧张→舒缓）？
- `#03` 结束：冲突是否已升级到最高点？
- `#04` 结束：真相是否给出了"解释框架"？
- 全文：对话占比是否在30%-40%之间？

### 5. 全局风格审查
加载 `references/style-guide.md`，逐项核对：

| 检查项 | 标准 | 方法 |
|--------|------|------|
| 对话比例 | 30%-40% | 统计对话行数/总行数 |
| 句子长度 | 常规25-40字，紧张10-20字 | 抽样检查各场景句长 |
| Bold使用 | 5-15处，仅用于关键信息 | 统计 `**` 出现次数 |
| 段落规模 | 平均1-5行 | 目测 |
| 口语语气词 | 至少出现3种不同语气词 | 搜索 嘛/呐/啊/呗/咋/啥 |
| 叙述者介入 | 至少3处第一人称评论 | 搜索"我""咱""你琢磨" |
| AI禁用词 | 0处 | 搜索 forbidden-phrases 中的一级禁令词 |
| **😀 幽默技巧** | **至少3种不同技巧，至少3处笑点** | 对照 style-guide 第七章逐一标记 |
| **🎢 情绪起伏** | **"三上三下"结构完整** | 对照 plot-structures 3.2节绘制曲线 |
| **🪝 节尾钩子** | **每个 #01 #02 #03 结尾有钩子** | 检查每节最后2行 |

### 6. 反AI味清洗
- 加载 `assets/forbidden-phrases.md`，全文搜索一级禁令词，命中即删除/重写
- 加载 `assets/dialogue-examples.md`，检查对话是否口语化、有地域特色
- 加载 `assets/opening-templates.md`，确认开头格式符合模板
- 检查是否有"文艺腔"——任何连续2句以上的纯景物/心理描写 → 精简
- **新增**：检查是否有"无幽默段落"——任何连续500字以上无笑点/无语气词的段落 → 插入幽默

### 6.5 AI 味系统性检测（新增步骤）

> 此步骤在"反AI味清洗"之后、"终审"之前执行。目标是系统性地扫描生成文本中的 AI 写作痕迹。

加载 `assets/ai-detection.md`，逐项执行检测：

#### A. 快速扫描（自动化优先）
- [ ] **一级禁令词命中**：搜索 `悄然 蓦然 倏地 璀璨 绚丽 静谧 安详 弥漫 笼罩 涌起 不禁 不由得 情不自禁` → 命中即删
- [ ] **对称句式检测**：搜索连续两个结构相同的短句（字数相近、词性对应）→ 打碎其中一个
- [ ] **的字密度**：`的` 字占比 > 4% → 减少定语，用动词替代形容词
- [ ] **虚词密度**：`其实 只是 或许 也许 大概 似乎 仿佛 宛如` 出现次数 > 每千字 2 次 → 删减

#### B. 结构扫描（人工判断）
- [ ] **段落均匀度**：是否有 1 行段和 8 行段交替？还是每段都 3-4 行？
- [ ] **情绪分布**：笑点间隔是否过于均匀？有没有"紧绷区"（连续无笑点）和"释放区"（连续包袱）的明显区分？
- [ ] **叙事弧完整度**：是否有 1-2 处"毛边"（未完全回收的伏笔、未交代清楚的配角）？太完整 = AI味

#### C. 细节扫描（人工判断）
- [ ] **具体细节**：有没有至少 3 个具体的年份/地名/品牌/价格？全是泛指 = AI味
- [ ] **对话差异度**：各角色话轮长度是否明显不同？太均匀 = AI味
- [ ] **情感表达**：有没有"他感到很恐惧"之类的直给？有没有用行为间接表达？

#### D. AI 味评分
```
每命中一个 P0 问题：扣 2 分
每命中一个 P1 问题：扣 1 分
扣满 5 分 → 判定"AI味过重"，需重写命中段落
```

详见 `assets/ai-detection.md` 第六节"修复优先级"。

### 7. 终审

加载 `assets/quality-checklist.md`，按 **P0 → P1 → P2** 顺序逐项打勾：

**P0 级（不改则故事不成立）**：
- [ ] **规则一致性**：核心超自然规则一句话是什么？全篇有无偷偷改规则？
- [ ] **反转升级**：反转是升级还是降级？标题承诺兑现了吗？
- [ ] **怪异现象链**：每个怪事都服务主线吗？删掉任何一个怪事，主线受影响吗？
- [ ] **真实反应**：主角第一次接触异常有没有完整的"否认→困惑→求证→害怕"链？
- [ ] **代价**：主角付出了什么具体的、可见的代价？

**P1 级（严重影响体验）**：
- [ ] **信息释放**：有没有超过3句的连续解释？真相是一次倒出的吗？
- [ ] **高潮力度**：高潮是"做"出来的还是"说"出来的？能画成一格漫画吗？
- [ ] **结局**：危机解决是一步还是多步？有无"尝试→失败→再试"？
- [ ] **配角独立**：每个配角有自己的欲望吗？有没有至少拒绝一次？
- [ ] **调查阻力**：主角获取真相的过程有没有至少一次失败？

**P2 级（精细化）**：
- [ ] **留白**：民俗解释是否不超过3句？有没有留30%让读者自己拼？
- [ ] **慢恐惧**：恐怖场景前有没有2-3句环境/感官描写？
- [ ] **幽默时机**：恐怖段落内有没有网络梗？搞笑有没有在恐怖"发酵"后才插入？
- [ ] **结尾**：有没有AI味金句总结？故事最后一段是具体画面还是抽象道理？
- [ ] **口吻内化**："你琢磨琢磨"是否超标？对话是否像真人说话？

**终审通过条件**：P0 全绿 + P1 ≤1项未通过 + P2 ≤3项未通过。

## 风格总纲（摘要）

> 完整版见 `references/style-guide.md`

### 叙事
- **视角**：第一人称框架（杨湃茶馆叙事）+ 第三人称故事主体，视角切换用空行分隔
- **时态**：框架用现在时，故事主体用过去时
- **开头**：`大家好，我是杨湃。` + `**有话要说**` 铺垫 → `**#01**` 进入故事
- **结尾**：`END`（独占一行）→ `> **奇闻说今古·兮香话鬼狐。下期我们不见不散。**`

### 语言
- **句长控制**：常规叙述25-40字/句；紧张时刻10-20字短句；关键转折处可50-80字复合句
- **段落**：平均1-5行，单句成段频繁使用
- **口语标签**：那么说、可谁承想、你琢磨琢磨、好家伙、列位、咱单说、就这么着
- **Bold**：仅用于 `**#01**` 章节标记、关键地名/人名、揭示性信息、金句对仗。全篇5-15处
- **对话**：占比30%-40%，嵌入叙述中不独立成段，用`：`引出。鬼怪说话要有"人味儿"

### 修辞
- 排比断句（最高频）
- 设问自答（"那么说……原来……"结构）
- 口语化比喻（"给那羊薅的跟葛优似的"）
- 夸张制造幽默（"这一宿，医院改奥运会场地了"）

### 情感曲线
```
强度
 8 |            ╱╲
 7 |          ╱    ╲___
 6 |   ╱╲___╱          ╲
 5 | ╱                    ╲___
 4 |/                          ╲_
 3 |                              ╲__  <- END 回落到温暖/哲理
   +-----------------------------------
    开头   #01    #02    #03   #04  收束
```

## 质量标准

### 必须满足（硬性指标）
- ✅ 开头200字内抛出一个"勾子"（悬念/画面/情感张力/幽默）
- ✅ 结尾提供情感满足或有力的留白
- ✅ 对话占比30%-40%，且每个说话者有不同的口头禅
- ✅ **全篇至少3处能让读者笑出声的幽默段落**（← 升级：从"至少1处"改为"至少3处"）
- ✅ **至少使用3种不同的幽默技巧**（自嘲/荒诞夸张/恐怖急转弯/破墙/方言梗/跨界混搭/谐音梗/黑色幽默 → 见 style-guide 第七章）
- ✅ **最恐怖的场景后必须跟一个搞笑缓冲**（技巧3：恐怖-搞笑急转弯，强制不可省略）
- ✅ **情绪曲线呈"三上三下"结构**（见 plot-structures 3.2节，强制不可省略）
- ✅ **每个 #01 #02 #03 结尾必须有钩子**（不完全信息钩/假安全感钩/冲向悬崖钩）
- ✅ 鬼怪/灵异现象至少给出一个"解释框架"（科学解释/民俗解释/至少暗示有解释）
- ✅ 至少1处"细思极恐"的细节
- ✅ 至少1处"假线索"让读者误判后再反转（← 新增）
- ✅ 全篇0处AI禁用词（见 `forbidden-phrases.md` 一级禁令）

### 应该满足（软性指标）
- ✅ 情感曲线符合"三上三下"形状
- ✅ 主角经历了"从疑到信"或等效认知改变
- ✅ 至少2处读者意想不到但合情合理的转折（← 升级：从"1处"改为"2处"）
- ✅ 有具体的地域/年代锚定细节（不是泛泛的"某个小镇"）
- ✅ 杨湃个人形象突出——至少1处自嘲、1处打破第四面墙

### 禁止
- ❌ 纯恐怖无缓冲（每500字恐怖场景后必须有舒缓段落）
- ❌ 连续500字以上无幽默/无转折/无情绪变化（← 新增：不允许"平铺直叙区"）
- ❌ 鬼怪说书面语/文言文
- ❌ 机械降神式的"高人"突然登场
- ❌ 超过15行无对话连续叙述
- ❌ 纯景物描写连续3句以上
- ❌ 解释性对话（角色不应说出明显是为了读者理解才说的话）
- ❌ 幽默针对受害者/死者/无辜者（黑色幽默只允许针对制度、命运、凶手）

### 8. 导出 Word 文档（新增步骤）

> 此步骤在终审通过后执行。将 Markdown 故事转换为格式化的 .docx 文件，方便发布、打印或传给非技术用户。

#### 使用方式

```bash
python story-writer-skill/scripts/convert-to-docx.py <故事.md> [输出.docx]
```

**示例**：
```bash
# 输出到同目录、同文件名
python story-writer-skill/scripts/convert-to-docx.py 机井沉冤.md

# 指定输出路径
python story-writer-skill/scripts/convert-to-docx.py 机井沉冤.md ./output/机井沉冤.docx
```

#### Word 文档格式说明

| 元素 | 排版 | 字体 |
|------|------|------|
| 故事标题 | 居中、22pt、加粗 | 微软雅黑 |
| `#01` `#02` 章节标题 | 居中、14pt、加粗、上间距18pt | 微软雅黑 |
| 正文段落 | 首行缩进2字符、1.5倍行距、12pt | 微软雅黑 |
| `**加粗**` | 段落内加粗、12pt | 微软雅黑 Bold |
| `> 引用块` | 左缩进1.5cm、11pt、灰色 | 微软雅黑 |
| `END` | 居中、14pt、灰色加粗、上下间距 | 微软雅黑 |
| 页边距 | 上下左右各 2.5cm | — |

#### 依赖

需要 `python-docx` 库：
```bash
pip install python-docx
```

## 迭代与终身学习

当用户提供他们修改后的版本并指出差异时：
1. 输出"风格Delta分析"：词汇层面 → 句式层面 → 结构层面 → 隐性风格
2. 将差异转化为具体规则："原规则 → 新规则"
3. 建议更新到哪个 references 文件
4. 用户确认后写入对应文件，在文件头部更新日志中记录版本变更

### 过拟合预警
当连续3篇生成的故事出现以下情况时，主动提醒用户：
- 开头模式完全一样
- Bold使用位置雷同
- 情感曲线几乎重叠
- 建议：锁死核心规则（对话格式、框架结构、禁忌词），放开次要规则（开头模板选择、章节数、修辞密度）
