# 三层自动校验规则

> 更新日志：2026-06-26 | v1.0 | 新建：大纲阶段拦截逻辑bug的三层校验体系

---

## 使用说明

在**步骤二（情节骨架）完成后**强制执行三层校验。每层逐条过，任意一条不通过 → 标注问题 → 修正后重检 → 全部通过才能提交用户确认。

---

## 第一层：时间线校验

> 目标：确保故事中所有事件的时间顺序、间隔、因果关系自洽。

### 1.1 时间轴绘制（强制）

以"年/月/日"为粒度，列出故事涉及的全部关键事件，按时间先后排列：

```
时间轴
━━━━━━━━━━━━━━━━━━
YYYY.MM.DD  事件A：[简述]
YYYY.MM.DD  事件B：[简述]（距A X天/月/年）
YYYY.MM.DD  事件C：[简述]（距B X天/月/年）
...
YYYY.MM.DD  故事现在时
━━━━━━━━━━━━━━━━━━
```

### 1.2 时间间隔合理性（逐条核验）

| # | 检查项 | 判定 | 常见Bug示例 |
|:--:|--------|:--:|------------|
| 1 | 相邻事件间隔是否与人物行为速度匹配？ | | 凶手三天内跨三省作案，交通工具对不上 |
| 2 | "三年后""五年后"等大跨度跳跃是否有中间状态的交代？ | | 人物三年没有任何变化，行为/语言/社会关系完全静止 |
| 3 | 关键物件的流转时间是否与人物行为时间吻合？ | | 一件古董三年转六手——每手平均半年，在古玩圈是否合理 |
| 4 | 信息传递速度是否符合时代通讯条件？ | | 90年代跨省追查，没有手机/互联网，靠蹲守黄页——时间成本是否体现在故事里 |
| 5 | 尸体/物证的腐化/老化程度是否与环境条件匹配？ | | 戈壁干燥→遗体三年未完全腐化✓；江南潮湿→遗体三年完好✗ |

### 1.3 时间线自检清单

```
□ 是否列出了完整的事件时间轴？
□ 每个跨年度跳跃是否有合理的人物状态变化？
□ 物件流转频率是否符合行业常态？
□ 通讯/交通耗时是否符合故事设定的年代？
□ 物理证据的保存/腐化状态是否匹配环境？
```

## 第二层：人物动机校验

> 目标：确保每个角色的行为有充分的内在驱动力，不出现"剧情需要所以这样做"的强行推进。

### 2.1 动机三要素（每个关键行为逐一核验）

| 要素 | 定义 | 缺失示例 |
|------|------|----------|
| **内驱力** | 角色为什么想做这件事？（欲望/恐惧/执念） | 老韩为什么敢把带证据的铜佛卖给熟人？→没有合理解释=违规 |
| **触发条件** | 为什么是这个时候做，不是之前/之后？ | 三年不卖，突然缺钱就卖——缺钱的原因具体吗？（赌博欠债✓ / "急用钱"模糊✗） |
| **阻力与代价** | 做这件事有什么风险？角色为什么仍然选择做？ | 陈国栋为什么不报警而选择民间蹲守？→必须有"报过警被驳回"的具体场景 |

### 2.2 动机合理性分级

| 等级 | 标准 | 判定 |
|:----:|------|:--:|
| **充分** | 内驱力+触发条件+阻力代价，三要素全部具体可落地 | ✅ 通过 |
| **可接受** | 三要素齐全但部分依赖读者推断 | ⚠️ 需补充至少一处暗示 |
| **不足** | 三要素缺一，或某一要素依赖巧合/强行设定 | ❌ 必须修改 |

### 2.3 常见动机漏洞

| 类型 | 表现 | 修复方向 |
|------|------|----------|
| **凶手降智** | 精明罪犯做出低级失误（如保留关键证据三年） | 补充其不知道证据存在的理由，或曾尝试但失败 |
| **受害者被动** | 受害者无任何自保行为，纯粹等待被发现 | 补充其自保尝试（如陈国良把证据藏佛像=主动防御） |
| **追查者盲目** | 追查者明明有更合理途径却不使用 | 补充其尝试过但被阻断的经历 |
| **旁观者全知** | 配角莫名知道不该知道的信息 | 删除或补充信息来源 |

### 2.4 人物动机自检清单

```
□ 每个角色的每个关键行为，是否都满足"内驱力+触发条件+阻力代价"三要素？
□ 是否有角色因为"剧情需要"而做出不符合其人设的行为？
□ 每个角色知道的信息，是否有明确的信息来源？
□ 反派/凶手的失误是否有合理解释（而非强行降智）？
```

## 第三层：信息来源校验

> 目标：确保故事中每个角色掌握的每条信息，都有可追溯的来源。杜绝"凭空知道"。

### 3.1 信息溯源表（每个关键信息逐一填写）

| 信息内容 | 掌握者 | 获取方式 | 获取时间 | 合理性 |
|----------|--------|----------|----------|:--:|
| 铜佛在刘一店里 | 陈国栋 | 跟踪老韩亲眼所见 | 1996.10 | ✅ |
| 马文忠=韩文忠 | 陈国栋 | 从醉掮客口中套出 | 1995年 | ✅ |
| 陈国良取了八万现金 | 陈国栋 | 通过汕头朋友搞到银行存根 | 1995年 | ✅ |
| 铜佛半夜嗡嗡响 | 刘一 | 亲耳听到 | 1996.10 | ✅ |

### 3.2 信息合理性分级

| 等级 | 标准 | 判定 |
|:----:|------|:--:|
| **可靠** | 信息来源明确+获取方式符合时代/人设+时间线自洽 | ✅ |
| **存疑** | 信息来源模糊（如"打听到""据说"）但无逻辑矛盾 | ⚠️ 尽量补充来源 |
| **违规** | 无任何来源交代，或获取方式明显不可能 | ❌ 必须修改 |

### 3.3 常见信息来源漏洞

| 类型 | 表现 | 修复方向 |
|------|------|----------|
| **天降情报** | 角色忽然知道某个关键信息，前面没有任何获取过程的描写 | 补一段他蹲守/打听/花钱买消息的细节 |
| **跨时空全知** | 90年代的人知道00年代才公开的信息 | 核对时代背景，替换为当时可获取的信息 |
| **陌生人精准定位** | 外地人第一次到某个城市，精准找到某个具体的人/店 | 补跟踪过程/黄页查号/本地人指点 |
| **陈婆全知** | 陈婆莫名知道精确时间/人名/事件细节 | 删精确数字，改为感受/暗示 |

### 3.4 陈婆信息权限特别规则

陈婆的信息来源必须约束在以下范围：
- ✅ 亲眼所见（如当天在店里看到老韩）
- ✅ 多年经验判断（如"这人走了三年了"——看铜佛氧化程度）
- ✅ 坊间传闻（如"听街上人说的"）
- ❌ 精确数字（死亡时间/金额/名字）
- ❌ 只有当事人知道的私密细节
- ❌ 一句"我看出来的"解释一切

### 3.5 信息来源自检清单

```
□ 每条关键信息是否都有明确的获取方式和获取时间？
□ 是否有角色知道了他不可能知道的事情？
□ 陈婆的信息是否全部约束在"所见/经验/传闻"范围内？
□ 跨省/跨地域的信息传递是否有合理的渠道和时间成本？
```

---

## 校验流程

```
步骤二完成 → 第一层时间线校验 → 不通过：修正 → 通过
           → 第二层人物动机校验 → 不通过：修正 → 通过
           → 第三层信息来源校验 → 不通过：修正 → 通过
           → 提交用户确认
```
